### Checking and Freeing Ports

**macOS / Linux — check what's on a port:**
```bash
lsof -i :4050    # Replace with any port number
```

**Kill whatever is using a port:**
```bash
# Find the PID first
lsof -ti :4050

# Then kill it
kill -9 $(lsof -ti :4050)
```

**Check all Organized-related ports at once:**
```bash
lsof -i :4050 -i :6006 -i :8000 -i :9099 -i :4000 -i :8080 -i :9199
```

**Windows:**
```powershell
netstat -ano | findstr :4050
# Then kill by PID:
taskkill /PID <pid> /F
```

> **Tip:** If `npm run dev` starts but the browser shows a blank page or connection refused, the port conflict is the first thing to check — especially port 8000 if another local server (e.g., a different Node project) is already using it.

## Quick Start (TEST Mode — No Backend)

```bash
# 1. Clone and install
git clone https://github.com/sws2apps/organized-app.git
cd organized-app
npm install

# 2. Copy environment file and enable TEST mode
cp .env.example .env
# Add this line to .env:
# VITE_APP_MODE="TEST"

npm run dev

# 4. Storybook (port 6006)
npm run storybook
```

## Development Modes

Organized has **two fundamentally different development modes** controlled by `VITE_APP_MODE` in your `.env` file.

### Mode 1 — TEST (Dummy Data, No Backend)

Set in `.env`:
```env
VITE_APP_MODE="TEST"
```

This mode renders `<DemoStartup />` instead of the real `<Startup />` auth flow. The app launches with pre-filled dummy congregation data — no Firebase, no backend, no login required.

<table>
  <thead>
    <tr><th>Behavior</th><th>Details</th></tr>
  </thead>
  <tbody>
    <tr><td>Auth</td><td>Skipped — <code>DemoStartup</code> pre-fills data immediately</td></tr>
    <tr><td>Firebase</td><td>Not needed</td></tr>
    <tr><td>Backend API</td><td>Not needed</td></tr>
    <tr><td>Sync worker</td><td>Disabled</td></tr>
    <tr><td>JW.org auto import</td><td>Disabled</td></tr>
    <tr><td>Onboarding wizard</td><td>Hidden</td></tr>
    <tr><td>Visual indicator</td><td>Blue <strong>"TEST"</strong> banner in the navbar</td></tr>
    <tr><td>Source constant</td><td><code>isTest = true</code> — in <code>src/constants/index.ts</code></td></tr>
  </tbody>
</table>

**When to use:** UI development, styling work, component prototyping, building features that don't involve sync or auth.

Equivalent to the live demo at **https://test.organized-app.com**.

---

### Mode 2 — Full Local Backend (Real Auth + OTP)

Remove `VITE_APP_MODE` from `.env` (or omit it entirely):

```env
VITE_FIREBASE_APIKEY=organized-local-api-key
VITE_FIREBASE_AUTHDOMAIN=localhost
VITE_FIREBASE_PROJECTID=organized-local
VITE_FIREBASE_APPID=organized-local-app-id
VITE_FIREBASE_AUTH_EMULATOR_HOST=http://127.0.0.1:9099
VITE_BACKEND_API=http://127.0.0.1:8000/
```

The full real `<Startup />` auth flow runs, connecting to local Firebase Emulators and the local `sws2apps-api` backend.

<table>
  <thead>
    <tr><th>Behavior</th><th>Details</th></tr>
  </thead>
  <tbody>
    <tr><td>Auth</td><td>Real Firebase Emulator auth (port 9099). Email login sends OTP to emulator — click the link shown in the emulator UI or terminal, no real email needed.</td></tr>
    <tr><td>Firebase</td><td>Firebase Emulator suite must be running</td></tr>
    <tr><td>Backend API</td><td><code>sws2apps-api</code> must be running on port 8000</td></tr>
    <tr><td>Sync worker</td><td>Active — background data sync works end-to-end</td></tr>
    <tr><td>Onboarding</td><td>Full congregation create/join flow available</td></tr>
    <tr><td>Visual indicator</td><td>No banner</td></tr>
    <tr><td>Source constant</td><td><code>isTest = false</code></td></tr>
  </tbody>
</table>

**When to use:** Auth flows, sync logic, role gating, congregation join workflow, backup/restore, any work touching `src/services/`, `src/wrapper/`, or `src/features/app_start/`.

**Full setup steps (from [LOCAL_ENVIRONMENT_SETUP.md](https://github.com/sws2apps/organized-app/blob/main/LOCAL_ENVIRONMENT_SETUP.md)):**

```bash
# --- Backend (sws2apps-api repo) ---
git clone https://github.com/sws2apps/sws2apps-api.git
cd sws2apps-api
npm install

# Copy required config files
cp storage.rules.example storage.rules
cp .env.example .env
cp .firebaserc.example .firebaserc

# Terminal 1: Start Firebase Emulators
npm run start:emulators

# Terminal 2: Start the API server
npm run dev

# --- Frontend (organized-app repo) ---
cd ../organized-app
npm install
cp .env.example .env
# (Do NOT set VITE_APP_MODE for full mode)

# Terminal 3: Start the frontend
npm run dev
```

**About the OTP login link:** After entering your email on the login screen, the email is not sent to a real address. Instead, a clickable link appears **directly on the emulator's web page** (`http://localhost:4000`). Click it there to complete authentication — no real email inbox needed.

> **Note:** Your current `.env` file omits `VITE_APP_MODE`, so this workspace currently runs in **full local backend mode**.
