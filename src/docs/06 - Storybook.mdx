# 6. Storybook Guide

Storybook is used to develop UI components in isolation, test them across all 8 themes, and document their API.

## Running Storybook

```bash
npm run storybook
```

Available at: `http://localhost:6006/`

## Story File Conventions

Stories live next to the component they document:

```
src/components/button/
├── index.tsx             # Component
├── index.types.ts        # Props type
└── index.stories.tsx     # Stories
```

## Writing a Story (CSF3)

We use the **Component Story Format 3** with `satisfies Meta<T>` for full type safety:

```tsx
import type { Meta, StoryObj } from '@storybook/react';
import Button from './index';

const meta = {
  title: 'Components/Button',
  component: Button,
  parameters: {
    layout: 'centered',
  },
  tags: ['autodocs'],
} satisfies Meta<typeof Button>;

export default meta;
type Story = StoryObj<typeof meta>;

export const Default: Story = {
  args: {
    variant: 'main',
    children: 'Click me',
  },
};

export const Secondary: Story = {
  args: {
    variant: 'secondary',
    children: 'Cancel',
  },
};
```

## Testing Themes in Storybook

Our app has 8 themes. When developing visual components, verify them across at minimum:
- `blue-light` (default)
- `blue-dark`
- One other color (e.g., `green-light`)

You can add a `globals` config in `.storybook/preview.ts` to allow theme switching via the toolbar.

## Tips

- Tag stories with `['autodocs']` to auto-generate a documentation page from JSDoc and TypeScript types.
- Use `parameters.layout: 'centered'` for small UI elements, `'fullscreen'` for full-page layouts.
- If a component requires context (Jotai provider, i18n, MUI theme), set it up in `.storybook/preview.tsx` decorators.
