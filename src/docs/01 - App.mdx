# 1. About the Organized App

**Organized** is a free, open-source Progressive Web Application (PWA) built for Jehovah's Witnesses congregations to manage assignments, meetings, ministry, and reports. It is cross-platform, secure, and available offline-first.

---

## Core Concepts

- **Local-first Architecture:** All data is primarily stored on the user's device using IndexedDB (via Dexie.js). The app is fully functional without internet. When connectivity is available, data syncs through Firebase using end-to-end encryption — unreadable even by the developers.
- **Role-based Access:** The app supports many user roles (admin, elder, ministerial servant, group overseers, pioneers, publishers, students, pocket users). Every feature is gated by the `useCurrentUser()` hook that derives permissions from Jotai atoms.
- **Open Source:** Publicly auditable at [github.com/sws2apps/organized-app](https://github.com/sws2apps/organized-app).

## Project Structure

```
src/
├── components/       # Global, reusable, generic UI components
├── features/         # Domain-specific feature modules
│   ├── meetings/     # Schedule editors, autofill, weekly view, exports
│   ├── persons/      # Person management, speakers catalog, assignments
│   ├── reports/      # Field service, attendance, publisher records
│   ├── ministry/     # Timer, personal reports, pioneer stats
│   └── congregation/ # Access management, field service groups, settings
├── hooks/            # Global custom React hooks
├── states/           # Jotai atoms separated by domain
├── pages/            # Top-level route pages
├── views/            # PDF/print templates (S-140, S-89, S-21, S-88)
├── layouts/          # App shell (navbar, bottom menu, root layout)
├── services/         # Firebase & API service wrappers
├── global/           # Global CSS design tokens
├── wrapper/          # Top-level providers (database, modal, feature flags, web worker)
└── docs/             # This Storybook documentation
```

## Congregation Account Lifecycle

1. Admin elder creates congregation in app → sets the encryption key.
2. Other users receive one-time invitation codes from the admin.
3. Users join and have roles assigned by admin.
4. All congregation data syncs E2E encrypted through Firebase.
